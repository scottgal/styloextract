Configuration Reference

This page documents every option understood by the extraction pipeline. It is the longest single page in the documentation and exists to keep the formatting pipeline honest under mixed-content load — prose, fenced code, inline links, ordered and unordered lists, plus a couple of small tables.

Service registration

Register the extraction service in your Program.cs using one of the helper extensions. The recommended path is full pipeline with SQLite-backed template store because it ships with the dependencies you already have, and the only knob worth changing in production is the fast-path threshold.

var builder = WebApplication.CreateBuilder(args);
builder.Services.AddStyloExtract(opts => {
    opts.FastPathThreshold = 0.85;
    opts.SlowPathThreshold = 0.75;
});

var app = builder.Build();
app.UseStyloExtract();
app.Run();

Each helper composes a set of defaults that suit a different deployment posture. See the table below for the deltas.

Profiles

Pass an ExtractionProfile per request to control what the renderer emits. The profile is orthogonal to detection — it controls projection, not classification.

  • MainContentOnly: emits MainContent, Article, Heading, Summary, Table, CodeBlock, and RepeatedItem roles. Drops navigation, chrome, ads.
  • RagFull: emits everything except footers, headers, ads, cookie banners, boilerplate, and the navigation roles.
  • AgentNavigation: the inverse of MainContentOnly: emits navigation, breadcrumb, and form roles for an agent that needs to walk the site.
  • DebugFull: emits everything with diagnostic comments above each block.

Thresholds

Two thresholds control the match path each request takes.

  1. The fast-path threshold gates whether a previously-seen layout signature counts as a hit.
  2. The slow-path threshold gates whether a similar-but-not-identical layout counts as a same-template observation for refit purposes.
  3. When neither threshold is met the request enters the novel-ephemeral path and induces a fresh extractor for the next observation.

Drift and refit

When the rule miss-ratio for a cached extractor exceeds the configured ratio, the orchestrator bug-outs and refits. See the refit guide for the math behind the EWMA decision.

If your RulesMissed divided by RulesApplied + RulesMissed exceeds 0.7 for three consecutive observations, the cached extractor is replaced.
