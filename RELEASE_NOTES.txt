StyloExtract 1.8.0 - 2026-06-24
================================

The ML release.

Closes the arbitrary-site coverage gap: hosts whose HTML the
heuristic classifier can't induce a clean template from (Shopify
themes, custom-CSS marketing pages, anything with class names
absent from framework-content-class-hints.json) now get LLM-driven
template induction in a background coordinator. The mechanism:

  * Novel template seen by LayoutExtractor → cache the heuristic-
    induced version → enqueue an enrichment job.
  * Background TemplateEnrichmentCoordinator drains the queue,
    calls an LLM with a slim DOM skeleton (1-4 KB), gets back YAML
    in the exact operator-template schema, writes it to the
    operator-template root.
  * The existing file-watching YamlFileOperatorTemplateStore picks
    up the YAML; the next request for that host hits the hard-
    override path with the LLM-induced template.

Runtime hot path is unchanged. The LLM never blocks a request.
Per-host cooldown + bounded queue + age-out + MainContent quality
gate. Hand-authored operator templates always win over induced ones.

Operator install (four DI calls):

    services.AddStyloExtract();
    services.AddStyloExtractOperatorTemplates("config/templates");
    services.AddOllamaTextProvider(o => o.Model = "gemma4:12b-it-qat");
    services.AddStyloExtractLlmInducer("config/templates");

New packages
------------

  Mostlylucid.StyloExtract.Ml — feature extractor + skeleton renderer
                                (AOT-clean; reusable for the
                                response-parser family).
  Mostlylucid.StyloExtract.Llm.Ollama — OllamaTextProvider over
                                /api/chat with streaming disabled.
                                Default model gemma4:e4b-it-qat
                                (Apache 2.0).

Surfaces
--------

  CLI:
    stylo-extract template dump-skeleton --url … | --file …
        Prints the slim representation the LLM would see for a page.
    stylo-extract template induce --url … --model gemma4:12b-it-qat
                                  [--write] [--root config/templates]
        Runs the LLM and prints (or persists) the resulting YAML.

  REST:
    POST /api/styloextract/templates/{host}/induce
        Body: { html, url? } → text/yaml on success. Same SSRF guard
        the existing /test endpoint shipped with.

Backend abstraction
-------------------

ILlmTextProvider in Mostlylucid.StyloExtract.Abstractions is the
backend contract. Default implementation is the Ollama provider in
Mostlylucid.StyloExtract.Llm.Ollama; operators can wire any
LLM client (cloud, local LlamaSharp, etc.) by implementing the
same one-method interface. The same provider will serve future
response parsers (PII redaction, content-safety verifier, etc.) per
the response-parser-family direction in
docs/ml-classifier-v2-design.md.

Model recommendation
--------------------

Gemma 4 E4B (qat, 6.1 GB, Apache 2.0) is the design's default for
the slow-path coordinator. Gemma 4 12B (qat, 7.2 GB) is the
preferred choice when memory allows; stronger structured-output
quality on the HTML-into-selectors task per Google's published
benchmarks. ONNX deployment is documented future work pending
onnxruntime-genai support for Gemma 4 (Microsoft tracking issue
#2062).

Real-world validation: ran the live-Ollama integration test against
locally-installed gemma4:e2b on the Acme product fixture. Output:

  host: acme.example
  rules:
    - role: MainContent
      selectors: [main.product-detail-root]
    - role: Header
      selectors: [header]
    - role: PrimaryNavigation
      selectors: [nav.site-nav]
    - role: Footer
      selectors: [footer.site-footer]

Four roles correctly identified in ~10 seconds on M5 CPU.

Compatibility
-------------

Backwards-compatible. Consumers who don't add the LLM helper see
v1.7.1 behaviour exactly: heuristic classifier + induced templates +
hand-authored operator templates. The Ollama package is opt-in.

Suite
-----

475 tests across 10 projects, all green. The 1.8.0 release adds
StyloExtract.Ml.Tests (20), Llm.Ollama provider tests (5), DI-
composition tests (3), live-Ollama integration tests (2 SkippableFact).
