StyloExtract

Layout-fingerprint matching with template-keyed extractor reuse for .NET 10.

Library (NuGet)

CLI binary

What it does

StyloExtract is not an HTML-to-Markdown converter. Generic tools like Trafilatura, Mercury, and Readability parse each page in isolation: they apply heuristics, discard boilerplate, and return text. That works, but it throws away something valuable: the fact that the same site usually produces the same page shape over thousands of requests. StyloExtract treats page structure as a fingerprint. The first time it sees a layout it induces an extractor (a set of CSS selectors and block roles) directly from the DOM. Every subsequent page that matches the same layout fingerprint reuses that extractor rather than re-running heuristics from scratch. The extractor is a learned centroid that drifts and refits as the site evolves. When the centroid drift crosses a threshold, the refit is recorded as a version event, giving you site-template-version monitoring as a side effect of extraction.

Two CLI flavors + library

stylo-extract (AOT)

  • Native binary, ~11 MB on arm64
  • No .NET runtime required
  • extract, export, import, monitor
  • File and HTTP sources
stylo-extract extract article.html
stylo-extract monitor --urls urls.txt \
  --store db.db --interval 00:30:00

stylo-extract-playwright

  • R2R self-contained, includes Playwright
  • All AOT subcommands plus:
  • --rendered flag for JS-rendered pages
  • install-browsers subcommand
stylo-extract-playwright extract \
  https://spa-example.com/page \
  --rendered

Library

  • AddStyloExtract() DI extension
  • ILayoutExtractor.ExtractAsync
  • AOT-compatible core stack
  • 11 StyloFlow signals per extract
var result = await extractor
  .ExtractAsync(html, uri);
Console.WriteLine(result.Markdown);
Console.WriteLine(result.Match.Status);
